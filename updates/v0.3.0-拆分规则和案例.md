## 为什么这样改

之前规则和多个案例放在同一个文件里，后续会越来越难查。

以后每次新增一个客户案例，就单独新建一个 `cases/xxx.md` 文件。这样 GitHub Desktop 的历史记录也会更清楚，能直接看到这次新增的是哪个案例。
