# 现场口播文案规则

## 目标

口播文案要让实体工厂、小公司、门店、加工厂老板听得懂，并且觉得这个问题和自己有关。

不要写成技术方案介绍，也不要写成宣传片。要像在客户现场调试时，顺手把问题讲清楚。

## 表达规则

1. 先展示问题，不要先讲概念。
2. 用“大家看一下”“我们现在这个地方”“另外一个办公室”这种现场语言。
3. 少说行业词，多说客户能看到的现象。
4. 不要直接说“老板痛点”“数字化转型”“高效协同”。
5. 要讲清楚原来为什么不行，现在怎么验证可以。
6. 结尾要落到实际效果，比如能打开、能修改、速度怎么样、稳定性怎么样。

## 推荐结构

```text
大家看一下，现在这个文件夹在办公室里面是可以打开的。
但是到了另外一个厂区，正常情况下它是打不开的。
因为两个地方不是同一个网络，文件共享只能在局域网里面用。

我们现在把两边网络打通以后，
另外一个办公室也可以像在总部一样打开这个文件夹。
你看这里可以打开文件，也可以修改，修改以后这边也能看到。

这种方案不是说能连上就行，
后面还要看速度、稳定性，还有员工电脑要不要一个个安装客户端。
```

## 禁止写法

```text
为客户打造跨地域数据协同平台。
帮助企业实现多办公区文件共享和 ERP 系统互联。
解决企业数据孤岛问题。
```

这些说法太官方，不像真实客户现场。
