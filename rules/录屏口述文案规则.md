# 录屏口述文案规则

## 定位

这个规则用于写录屏或手机拍摄时的口述稿。

口吻要像真实操作时顺手讲，不要像提前写好的讲解稿。内容可以有停顿、重复和转折，但逻辑要清楚。

## 核心口吻

可以这样写：

```text
我们这个文件共享你看一下，其实设置很简单。
但是呢，它能用是一回事，你要让它一直稳定地用，其实就没有那么简单。
很多时候不是你这个共享没设置好，而是每台电脑自己的状态不一样。
```

## 写作规则

1. 不要用正式开场。
2. 不要先介绍方案，先讲屏幕上正在看到的现象。
3. 可以用“其实、但是呢、你看这个地方、很多人会以为”这种口语。
4. 第一人称可以用，但不要一直强调“我做了什么”，重点放在判断过程。
5. 不要写得太完整，真实口述可以有一点重复。
6. 每一段最好都能对应到屏幕上的一个操作或一个设置。
7. 结尾要写验证动作，比如打开、保存、复制、刷新、换电脑再测。

## 禁用口吻

不要写成这种：

```text
今天介绍一个稳定访问文件的办法。
这个办法适合很多地方使用。
我们已经把这个问题处理好了。
```

这种太像摆拍，也太像包装稿。

## 推荐结构

```text
这个设置看起来很简单
但是长期用的时候经常出问题
先指出一个常见误判
再打开界面说明判断顺序
然后做一个实际测试
最后说明不能只看一次成功，要看后面状态
```

## 判断标准

写完以后读一遍，如果像正式讲课，就要改。

比较合适的状态是：

```text
像一个人在电脑前一边点设置，一边把自己踩过的坑讲出来。
```
