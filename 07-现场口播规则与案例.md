# 现场口播规则与案例

## 这类视频的定位

这类视频不是公司宣传片，也不是技术教程。

它更像一个现场调试师傅，拿着客户现场的电脑、文件夹、系统界面，一边操作一边讲：

```text
这个问题客户现场经常遇到。
这个地方为什么原来不行。
我们现在怎么处理。
处理完以后现场打开给你看。
速度怎么样，能不能改，能不能保存。
```

目标观众是做工厂、喷绘、加工、门店、项目部等实体业务的人。语言要像平时说话，不要像 PPT。

## 以后写文案要遵守的规则

### 1. 先拿现场东西说话

开头不要先讲概念。

不要这样：

```text
多办公区统一资料管理，是企业数字化的重要环节。
```

要这样：

```text
大家看一下，这边就是客户办公室的一台电脑，这个文件夹在他们本地是可以共享的。
```

### 2. 不要用太大的词

少用这些词：

```text
企业级
数字化
统一管理
业务协同
交付体系
解决方案闭环
赋能
提效
```

可以换成：

```text
能不能打开
会不会卡
会不会断
员工会不会用
换电脑麻不麻烦
这边改了那边能不能看到
今天能用，过几天还稳不稳定
```

### 3. 不要一直说“老板”

可以说：

```text
很多做厂的朋友
很多做喷绘的客户
很多办公室
客户现场
他们这边
另外一个办公区
车间那边
```

少说：

```text
老板关心
老板需要
老板愿意付费
```

这些可以在内部分析时用，对外口播不要一直讲。

### 4. 一条视频只讲一个现场问题

不要一条视频同时讲 ERP、群晖、权限、备份、客户管理。

一条视频只讲一个点：

```text
异地办公室怎么打开总部共享文件夹。
异地办公室怎么访问总部 ERP。
为什么不用每台电脑都装客户端。
两个办公室文件修改后能不能互相看到。
```

### 5. 必须有演示动作

视频里尽量要有动作，而不是全程讲道理。

可以演示：

```text
打开共享文件夹
打开 ERP
新建一个文件
修改一个文件
保存文件
回到另一台电脑查看
看传输速度
看打开是否卡顿
拔掉或切换网络后恢复测试
```

## 通用口播结构

```text
第 1 段：大家看一下现场是什么
第 2 段：原来为什么不方便
第 3 段：这个问题很多客户会遇到
第 4 段：我们现在怎么处理，不讲太深
第 5 段：现场打开测试一下
第 6 段：看结果，能打开、能改、能保存
第 7 段：这个方案适合哪些情况
```

## 案例背景

客户行业：

```text
喷绘、广告制作
```

客户现场情况：

```text
第一个办公区有 ERP 系统和文件资料。
另一个办公区也要访问 ERP 和文件。
文件可能放在群晖或共享文件夹里。
客户不想每台员工电脑都安装组网客户端。
我们做了两端组网。
第二个办公区通过一台电脑或设备做网关。
员工电脑通过这个网关访问第一个办公区的 ERP 和文件。
```

对外表达重点：

```text
两个办公区像在同一个局域网里一样使用。
员工电脑不用每台都折腾。
能打开 ERP。
能打开总部文件。
能修改保存。
关键是要长期稳定。
```

## 案例文案 1：异地打开同一个共享文件夹

标题：

```text
两个办公室不在一起，怎么打开同一个文件夹？
```

口播：

```text
大家看一下，这边是客户办公室里共享出来的一个文件夹。

如果几台电脑都在同一个办公室、同一个路由器下面，访问这个文件夹一般没什么问题。

但是他们还有另外一个办公区，那边不是这个网络，直接这样访问肯定是打不开的。

很多做喷绘、做工厂的客户都会遇到这种情况，一个地方放文件，另外一个地方也要用。

如果每次都靠微信发，或者拷 U 盘，文件一多就很乱，改了哪个版本也不好确认。

我们现在做的是把两边网络打通，让另外一个办公区也能像在这边一样访问这个文件夹。

大家看一下，这是另外一个办公区的电脑，我们现在直接打开这个共享地址。

可以看到文件夹已经出来了，里面这些文件也是能正常打开的。

我们随便改一个文件，保存一下，再回到这边看。

这边也能看到刚才改过的内容。

这种方案不是说只能临时用一下，关键是要调到稳定，不然今天能打开，明天又断，那现场就没法用。
```

画面建议：

```text
共享文件夹
两个办公室示意
另一台电脑打开共享路径
打开文件
修改保存
回到原电脑查看
```

## 案例文案 2：异地访问 ERP

标题：

```text
新办公区也要用总部 ERP，怎么处理？
```

口播：

```text
大家看一下，这个客户是做喷绘的，他们第一个办公区这边有一套 ERP 系统。

平时接单、查订单、看客户资料，很多东西都在这个系统里面。

现在他们又有一个新的办公区，那边员工也要进这个 ERP。

如果直接在外面访问，很多时候不是打不开，就是不稳定，或者还要每台电脑单独装东西。

我们这边给他处理的方式，是把两个办公区的网络打通。

打通以后，另外一个办公区的电脑，就可以像在这边办公室一样访问这个 ERP。

大家看一下，现在我在另外一个办公区这台电脑上打开。

ERP 这个登录界面已经能出来，进去以后查询订单也是正常的。

然后文件这块也一起测试一下，客户资料、喷绘文件这些也可以从总部这边打开。

这种场景其实很多小工厂、小加工厂都会遇到。

不是说一定要换系统，很多时候是原来的系统还能用，只是要把异地访问这块处理好。
```

画面建议：

```text
ERP 登录界面
订单查询页面
另一办公区电脑
总部文件夹
客户资料或喷绘文件
```

## 案例文案 3：不用每台电脑都装客户端

标题：

```text
异地办公，不想每台电脑都装客户端怎么办？
```

口播：

```text
很多客户一开始做异地访问，都会想到一个办法，就是每台电脑都装一个客户端。

这个办法不是不能用，但现场电脑一多就麻烦。

今天这个员工换电脑了，要重新装；明天那台电脑系统坏了，又要重新弄。

有些员工也不懂这些东西，点错了、连不上了，还得一直找人处理。

所以这个客户现场，我们没有让每台电脑都单独折腾。

我们是在另外一个办公区做了一个统一的接入，让那边的电脑通过这个网关来访问总部。

大家看一下，这边员工电脑没有每台都去单独装组网客户端。

但是它现在一样可以打开总部的 ERP，也可以打开总部的共享文件夹。

这样现场用起来会简单很多。

员工只管正常打开文件、打开系统，后面的网络打通和稳定性，我们这边来处理。
```

画面建议：

```text
多台员工电脑
统一网关电脑或设备
ERP 打开
共享文件夹打开
员工正常操作
```

## 案例文案 4：测试速度和稳定性

标题：

```text
异地打开文件，不能只看能不能连上，还要看稳不稳。
```

口播：

```text
大家看一下，异地访问文件，不是说能打开一次就算做好了。

现场真正用的时候，员工每天都要打开、修改、保存文件。

如果一会儿能用，一会儿断开，或者打开一个文件等半天，那最后还是用不起来。

所以我们调完以后，一般会现场测一下打开速度和保存效果。

现在我这边从另外一个办公区打开总部这个文件夹。

大家可以看一下，文件列表出来得还是比较快的。

我们打开一个文件，再改一下内容，保存。

然后回到总部这边看，刚才改过的文件也能看到。

像这种方案，最重要的不是说听起来多复杂，而是现场员工每天用的时候要顺。

能稳定打开，能正常保存，后面才算真正能用。
```

画面建议：

```text
打开文件夹速度
文件列表加载
修改文件
保存文件
另一边查看结果
网络状态或传输速度
```

## 以后 AI 写这类文案的提示词

```text
你写的是现场口播文案，不是宣传片文案。

受众是小工厂、喷绘店、加工厂、实体门店的负责人。

语言要像现场师傅一边操作一边讲，口语化、真实、不装。

不要写大词，不要写企业级、数字化、赋能、统一管理。

少说老板，多说客户现场、这边、那边、员工电脑、办公室、车间、文件夹、ERP。

每条文案只讲一个具体问题。

必须有现场演示动作，比如打开文件夹、打开系统、修改保存、回到另一边查看。

请根据下面案例生成 3 条文案，每条 300 到 500 字：

案例：
【粘贴真实案例】
```
