# 录屏口述工作流

录屏文案不要写得太像稿子，要先按真实操作顺序整理。

## 第一步：写现象

先写一句真实现象。

```text
这个共享目录昨天还能打开，今天这台电脑突然进不去了。
```

## 第二步：写误区

说明很多人第一反应会看哪里。

```text
很多人一看到打不开，就先去改共享权限，其实这个不一定是权限问题。
```

## 第三步：写判断

写自己会按什么顺序判断。

```text
先看共享目录本身是不是正常，再看这台电脑能不能访问到那个地址，最后再看凭据和防火墙。
```

## 第四步：写操作画面

每句话尽量对应一个画面。

```text
这里先打开共享设置。
这里看一下网络类型。
这里打开凭据管理器。
这里换成 IP 再访问一次。
```

## 第五步：写验证

结尾不要说“已经解决”，要写怎么验证。

```text
现在目录能打开，文件能保存，换另一台电脑再进一次也正常。这样才算基本稳定，不是只打开一次就结束。
```

## 生成提示词

```text
请根据下面的问题卡，生成一段录屏口述稿。

要求：
1. 像边操作边说，不要像正式讲解。
2. 不要写开场口号。
3. 允许有“其实、但是呢、你看这个地方”这种口语。
4. 先讲现象，再讲误区，再讲判断，再讲验证。
5. 不要出现身份标签和具体领域标签。
6. 不要写成包装稿。
```
