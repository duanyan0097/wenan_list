# 整体方案

这个仓库用来沉淀录屏口述内容，不追求文案漂亮，重点是让内容听起来像真实操作。

适合整理的主题包括：

- Windows 共享为什么看起来简单，但用起来不稳定。
- 同一个共享目录为什么有的电脑能打开，有的电脑打不开。
- 异地访问为什么不能只改共享权限。
- 网关方式为什么能减少每台电脑反复维护。
- 文件同步、备份、权限和恢复应该怎么验证。
- 调完以后怎么判断状态是否真的正常。

## 核心目标

1. 把真实问题讲成普通人能听懂的话。
2. 把“设置步骤”背后的坑讲出来。
3. 让新人可以照着问题卡写出自然口述稿。
4. 让每条内容都能对应到屏幕操作或手机拍摄画面。

## 不追求的东西

不追求专业词堆得多。

不追求开场很完整。

不追求把每个问题讲成标准方案。

重点是讲清楚：

- 这个地方看起来简单在哪里。
- 实际容易出问题在哪里。
- 常见误判是什么。
- 应该先检查什么。
- 最后怎么测试状态。

## 推荐内容结构

```text
先说一个很常见的现象
再打开对应设置或界面
讲这个地方为什么容易误判
再说自己的判断顺序
最后做一个实际测试
```

## 录屏画面要求

每条内容最好能对应到真实画面：

- 文件夹共享设置。
- 网络类型和防火墙。
- 凭据管理器。
- 共享目录访问。
- 复制文件速度。
- 同步状态。
- 网关或连接状态。
- 系统登录和页面切换。
