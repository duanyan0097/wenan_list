# 整体方案

## 定位

这个资料库服务于一个具体业务方向：

```text
企业资料安全与客户交付管理
```

主要客户是实体工厂、中小企业、分公司办公、工程项目部、贸易公司、服务公司等。这些客户往往已经有群晖、威联通、Windows 共享、可道云、CRM、ERP、财务软件等系统，但信息分散、权限混乱、同步和备份边界不清。

## 真实业务目标

通过整理问题库，解决三个问题：

1. 找到老板真正关心的问题。
2. 让新人也能按固定框架挖掘客户痛点。
3. 为短视频文案、销售话术、客户方案提供稳定素材。

## 为什么不能只写技术问题

技术问题通常是：

```text
怎么配置同步？
怎么设置权限？
为什么文件不同步？
```

但老板真正关心的是：

```text
资料会不会泄露？
客户交付会不会延误？
员工离职会不会带走资料？
文件丢了能不能恢复？
出了问题能不能查到责任？
```

所以问题库必须把技术现象翻译成业务风险。

## 最终形成的内容资产

这个问题库以后可以输出：

- 短视频选题。
- 短视频分段文案。
- 客户沟通话术。
- 售前需求表。
- 部署方案模板。
- 客户交付管理系统需求。
- CRM 客户跟进内容。

## 与现有系统的关系

当前已有或计划中的系统包括：

```text
SoftEther 管理系统
CRM 客户交付管理系统
视频创作发布系统
可道云资料仓库
飞书多维表格历史数据
```

推荐关系：

```text
CRM / 客户交付系统 = 客户主数据中心
SoftEther 管理系统 = 服务管理模块
视频创作系统 = 内容生产和发布模块
可道云 = 客户资料文件仓库
问题库 = 内容和销售素材来源
```

不建议一开始把所有系统强行合并。更合理的是：客户数据统一，业务系统通过 API 或导入导出联通。
