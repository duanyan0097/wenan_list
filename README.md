# 文案问题库

这个仓库用于整理文件共享、异地访问、同步、备份、权限、网关和稳定性相关的录屏口述文案。

现在的核心方向不是写标准讲课稿，而是写成真实操作时顺手讲出来的内容。

## 先看这个文件

后续写文案先看：

```text
文案编写指导.md
```

这个文件是总指导。其他文件只是补充资料和案例。

## 文件规则

- `rules/`：只放口述风格、禁用口吻、案例整理规则。
- `cases/`：每个问题单独一个文件，方便后续按主题查找。
- `updates/`：每次版本更新单独一个文件，说明这次改了什么。
- `CHANGELOG.md`：只保留版本索引和简要摘要。

## 写法原则

文案要像拿手机或电脑录屏时边操作边说。

不要写成这种：

```text
今天介绍一个稳定访问文件的办法，适合很多场景使用。
```

要写成这种：

```text
我们这个文件共享你看一下，其实设置很简单。
但是它能用，和它长期稳定地用，是两回事。
很多时候不是你这个共享没点对，而是每台电脑的系统版本、网络类型、凭据缓存都不一样。
```
