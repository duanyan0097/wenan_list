# 文案问题库

这个仓库用于整理文件共享、异地办公、客户部署、到期维护、短视频选题和现场口播文案。

以后按这个结构保存资料：

```text
01-整体方案.md
02-老板问题库.md
03-问题挖掘渠道.md
04-问题卡模板.md
05-短视频文案工作流.md
06-资料来源.md
rules/
cases/
updates/
CHANGELOG.md
README.md
```

## 文件规则

- `rules/`：只放写作规则、口播规则、案例整理规则。
- `cases/`：每个真实案例单独一个文件，不要多个案例合并在一个文件里。
- `updates/`：每次版本更新单独一个文件，说明这次改了什么、为什么改。
- `CHANGELOG.md`：只保留版本索引和简要摘要。

## 使用方式

1. 先看 `01-整体方案.md`，理解这个问题库的定位。
2. 再看 `02-老板问题库.md`，挑选适合拍视频或销售沟通的问题。
3. 用 `04-问题卡模板.md` 把单个问题整理成标准问题卡。
4. 需要写现场口播时，先看 `rules/现场口播文案规则.md`。
5. 每次新增真实案例，都在 `cases/` 新建一个独立文件。

## 核心原则

文案要像真实现场调试，不要像宣传片。

不要写成这种：

```text
跨地域文件共享解决方案，帮助企业实现高效协同。
```

要写成这种：

```text
大家看一下，这个文件夹在办公室里面可以打开，但是到了另外一个厂区就打不开了。
这个问题不是员工不会用，而是两个地方的网络本来就不通。
我们现在把两个地方打通之后，在另外一个办公室也能像在总部一样打开这个文件夹。
```
